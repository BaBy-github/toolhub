# 实现工具页面骨架屏解决Footer闪烁问题

## 问题分析
当切换工具时，由于工具页面的内容（如编辑器）需要一定时间初始化，而footer是固定在页面底部的，导致footer会提前显示出来，造成页面布局闪烁。

## 解决方案
为工具页面添加骨架屏，在页面内容加载完成前显示占位内容，保持页面高度稳定，避免footer闪烁。

## 实现计划

### 1. 创建通用骨架屏组件
- 创建 `SkeletonLoader.vue` 组件，包含：
  - 页面头部占位
  - 工具栏占位
  - 编辑器区域占位
  - 使用 Tailwind CSS 的 `animate-pulse` 类实现加载动画
  - 参考 Tailwind UI Blocks 设计风格

### 2. 修改工具页面
为所有工具页面（ToDiff.vue, ToJson.vue, ToBase64.vue, ToXml.vue, ToEscape.vue）添加：
- 添加 `isLoading` 状态，初始为 `true`
- 在 `onMounted` 钩子中，当页面内容加载完成后将 `isLoading` 设为 `false`
- 使用 `v-if` 条件渲染骨架屏或实际内容

### 3. 骨架屏设计
- 页面头部：显示标题占位和返回按钮占位
- 工具栏：显示工具名称、描述和操作按钮占位
- 编辑器区域：显示高度为 `60vh` 的占位区域，与实际编辑器高度一致
- 使用 Tailwind CSS 的渐变和脉冲动画，保持与应用整体风格一致

### 4. 样式优化
- 确保骨架屏的高度与实际内容高度一致
- 使用响应式设计，适配不同屏幕尺寸
- 保持骨架屏与实际内容的视觉结构一致

## 预期效果
- 切换工具时，页面布局保持稳定，footer 不会提前显示
- 显示优雅的骨架屏加载动画，提升用户体验
- 与应用整体设计风格保持一致

## 技术栈
- Vue 3
- Tailwind CSS
- 参考 Tailwind UI Blocks 设计风格
