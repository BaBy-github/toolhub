# 将工具列表抽离成统一数据

## 问题分析
当前工具列表在代码中分散使用，包括：
- `Home.vue` 中硬编码的工具卡片
- `router/index.ts` 中的路由配置
- `ConversionButton.vue` 中的转换选项
- `i18n/locales/` 中的国际化文本

这种方式导致工具信息不统一，修改时需要多处同步更新，容易出错。

## 解决方案
将工具列表抽离成统一的数据格式，方便维护和更新。

### 2. 修改 Home.vue
使用工具数据渲染工具列表：


### 3. 修改 router/index.ts
使用工具数据生成路由配置：

```typescript
import { createRouter, createWebHistory } from 'vue-router'
import { tools } from '@/data/tools'

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    { path: '/', component: () => import('@/pages/Home.vue') },
    ...tools.map(tool => ({
      path: tool.path,
      component: () => import(`@/pages/${tool.name}.vue`)
    }))
  ],
})

export default router
```

### 4. 更新 ConversionButton 使用
确保 ConversionButton 组件能使用工具数据生成转换选项。

### 5. 保持国际化文本结构
国际化文本结构保持不变，只在工具数据中引用国际化键。

## 预期效果
- 工具信息统一管理，修改时只需更新一处
- 避免硬编码，提高代码可维护性
- 方便添加新工具，只需在数据中添加新条目
- 确保所有使用工具信息的地方保持一致
