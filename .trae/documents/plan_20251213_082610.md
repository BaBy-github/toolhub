### 实现计划

1. **添加错误状态管理**：在ToCrypto.vue组件中添加`error` ref来管理错误状态
2. **修改错误处理逻辑**：在`processInput`函数中，将错误信息存储到`error` ref中，而不是直接设置到输出中
3. **添加错误显示区域**：在输出编辑器下方添加专门的错误显示区域
4. **优化错误信息**：提供更友好的错误提示

### 具体修改

1. **添加error ref**：在组件的ref声明部分添加
   ```typescript
   const error = ref('')
   ```

2. **修改processInput函数**：
   ```typescript
   async function processInput() {
     const trimmed = input.value.trim()
     if (!trimmed) {
       output.value = ''
       error.value = ''
       return
     }
   
     try {
       if (isEncryptMode.value) {
         output.value = await encryptString(trimmed, cryptoSettings.value)
         error.value = ''
       } else {
         output.value = await decryptString(trimmed, cryptoSettings.value)
         error.value = ''
       }
     } catch (e) {
       output.value = ''
       error.value = e instanceof Error ? e.message : 'Unknown error'
     }
   }
   ```

3. **添加错误显示区域**：在输出编辑器下方（第413行后）添加
   ```vue
   <div v-if="error" class="border-t p-2 text-sm text-red-600">{{ error }}</div>
   ```

4. **修改watch input逻辑**：确保在输入变化时清空错误
   ```typescript
   if (showOutput.value && trimmed.length === 0) {
     showOutput.value = false
     error.value = ''
   }
   ```

### 预期效果

- 当解密失败时，输出编辑器为空
- 错误信息显示在输出编辑器下方，使用红色文字
- 与ToJson组件的错误显示方式保持一致
- 提供更友好的错误提示
