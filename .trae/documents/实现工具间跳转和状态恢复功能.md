# 实现工具间跳转和状态恢复功能

## 1. 状态管理设计
- 创建一个状态管理文件 `src/utils/toolState.ts`，用于保存工具间跳转的历史记录和编辑状态
- 实现 `pushToolState` 和 `popToolState` 函数，用于保存和恢复工具状态

## 2. 修改返回按钮逻辑
- 修改每个工具页面的 `goBack` 函数，从状态管理中恢复上一个工具的状态
- 移除当前清空输入的逻辑，改为回退到上一个工具或首页

## 3. 添加下一步按钮
- 在每个工具页面的输出区域添加下一步按钮
- 为不同工具设计合适的下一步跳转目标（如ToJson→ToDiff，ToXml→ToJson等）
- 点击下一步按钮时，保存当前状态并跳转到目标工具，同时将当前输出作为目标工具的输入

## 4. 实现工具间数据传递
- 在路由跳转时，通过状态管理传递当前工具的输出作为目标工具的输入
- 在目标工具页面加载时，从状态管理中获取并设置输入值

## 5. 适配不同工具
- 为每个工具页面添加相应的下一步按钮
- 确保不同工具间的数据格式兼容

## 6. 测试和优化
- 测试工具间跳转和状态恢复功能
- 优化用户体验，确保操作流畅

### 具体修改文件
1. `src/utils/toolState.ts` - 新增状态管理文件
2. `src/pages/ToJson.vue` - 添加ToDiff和ToXml下一步按钮
3. `src/pages/ToXml.vue` - 添加ToJson和ToDiff下一步按钮
4. `src/pages/ToDiff.vue` - 添加返回按钮状态恢复
5. `src/pages/ToBase64.vue` - 添加返回按钮状态恢复
6. `src/pages/ToEscape.vue` - 添加返回按钮状态恢复

### 实现细节
- 使用Vue的组合式API创建状态管理
- 为每个工具设计合适的下一步跳转逻辑
- 确保返回按钮能正确恢复上一个工具的编辑状态
- 保持代码的可维护性和扩展性
