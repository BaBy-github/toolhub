## 问题分析
1. 首次修复后，主题切换功能仍然不工作，只有在 macOS 系统级切换深色模式时才生效
2. 进一步研究发现，Tailwind CSS v4 中需要在 CSS 文件中使用 `@custom-variant` 指令来定义 dark 变体
3. 当前的 style.css 文件中缺少这个配置，导致 dark 类没有被正确识别和应用

## 解决方案
1. 在 `style.css` 文件中添加 `@custom-variant` 指令，定义 dark 变体
2. 确保 Tailwind CSS v4 正确识别和应用 dark 类
3. 测试主题切换功能

## 实施步骤
1. 编辑 `/Users/baby/Documents/IdeaProjects/toolhub/src/style.css` 文件
2. 在文件顶部添加 `@custom-variant dark (&:where(.dark, .dark *));` 配置
3. 重启开发服务器
4. 测试主题切换按钮是否正常工作

## 预期结果
- 右上角的主题切换按钮点击后，页面会在浅色和深色模式之间切换
- 切换状态会被保存在 localStorage 中
- 页面元素会正确应用对应的暗色主题样式
- 不再依赖系统级的深色模式设置
