# 修复To Diff工具的输入和对比功能

## 问题分析
1. **导入问题**：模板中仍然使用了`CodeEditor`组件，但脚本中只导入了`DiffEditor`，导致`CodeEditor`未定义
2. **UI设计问题**：当前实现是先输入再显示对比结果，而用户想要的是直接在左右两侧输入，实时显示对比结果
3. **交互体验问题**：需要优化用户体验，让用户可以直接在对比视图中输入和编辑

## 修复方案

### 1. 修复导入问题
- 修改脚本导入，同时导入`CodeEditor`和`DiffEditor`组件
- 确保模板中使用的组件都已正确导入

### 2. 优化UI设计
- 移除当前的"先输入再显示结果"模式
- 直接显示左右两侧的对比视图，让用户可以实时输入和编辑
- 保持与用户期望一致的交互方式

### 3. 配置DiffEditor组件
- 确保DiffEditor组件正确配置，支持实时对比
- 允许用户在左右两侧直接输入和编辑内容
- 提供清晰的差异高亮显示

## 实现步骤

### 1. 修改组件导入
```typescript
// 从
import { DiffEditor } from 'monaco-editor-vue3'
// 改为
import { CodeEditor, DiffEditor } from 'monaco-editor-vue3'
```

### 2. 重构UI布局
- 移除`showOutput`状态变量，不再使用条件渲染
- 直接显示DiffEditor组件，占据整个页面主要区域
- 提供清晰的左右侧标签，指示原始内容和修改后内容

### 3. 优化交互体验
- 确保DiffEditor组件支持实时编辑
- 提供复制、清空等便捷功能
- 保持与现有工具一致的UI风格

### 4. 测试验证
- 测试左右两侧是否可以正常输入
- 测试差异是否实时显示
- 测试复制、清空等功能是否正常工作

## 预期效果
- 页面直接显示左右两侧的对比视图
- 左侧显示原始内容，右侧显示修改后内容
- 支持在两侧直接输入和编辑
- 实时显示差异，高亮显示修改、添加和删除的内容
- 提供便捷的复制、清空等功能
- 保持与现有工具一致的UI风格和交互方式

## 技术要点
- 正确导入和使用`CodeEditor`和`DiffEditor`组件
- 配置DiffEditor组件支持实时编辑
- 优化UI布局，提供良好的用户体验
- 确保差异对比功能正常工作
